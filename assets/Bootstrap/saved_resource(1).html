                                                            Kartik V —

                                                            The easiest way to do this would be configure and override the elErrorContainer property which will override the default error container to the container element identifier of your choice. Thus default plugin error messages can be displayed in your own error container. You can then use the other plugin error events to run scripts / other error library plugins on this element.

                                                            Alternatively you can configure elErrorContainer to an element on your page which is hidden and then use your own error library to read the content in this container and display your error messages - via the plugin error events.
